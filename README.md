# 知径 Knowvia macOS Demo

知径 Knowvia 是知行星舱 StarCabin AI 旗下的 AI 增强型多源知识路径生成系统，帮助学习者从论文、网页、书籍、课程资料、笔记与可靠外部来源中形成可追溯、可校准、可学习、可输出、可持续生长的个人 Knowledge Pathway。

Slogan：让知识成为路径。  
English：Where knowledge becomes your path.

本仓库已完成早期 macOS 技术 Demo，并正在根据《知径 Knowvia 产品正式档案》切换到 Knowledge Pathway 主线。当前实现中的资料导入、阅读器、来源回溯、批注、知识卡片、轻量主题整理与导出能力将作为新 MVP 的基础层继续复用。

## 环境要求

- macOS 14+
- Xcode 26 或兼容版本
- Swift 6 工具链

## 运行方式

使用 Xcode：

1. 打开 `Knowvia.xcodeproj`。
2. 选择 `Knowvia` Scheme。
3. 运行 macOS App Target。

使用命令行构建：

```bash
xcodebuild \
  -project Knowvia.xcodeproj \
  -scheme Knowvia \
  -configuration Debug \
  -derivedDataPath DerivedData \
  CODE_SIGNING_ALLOWED=NO \
  build
```

运行测试：

```bash
xcodebuild \
  -project Knowvia.xcodeproj \
  -scheme Knowvia \
  -configuration Debug \
  -derivedDataPath DerivedData \
  CODE_SIGNING_ALLOWED=NO \
  test
```

## 当前能力

- 使用 `NavigationSplitView` 构建 Sidebar、Workspace、Inspector 三栏工作台。
- Dashboard 展示基础统计、导入入口和温和的学习路径占位组件。
- 资料库支持按钮导入和拖拽导入 `PDF`、`TXT`、`Markdown`。
- 资料库支持修改资料显示名称、编辑标签、删除 App 内的本地资料副本，并显示导入成功提示。
- 导入文件复制到 App 沙盒内的 `Application Support/KnowviaLibrary/<UUID>/`。
- 资料元数据使用 SwiftData 保存在本机。
- Sidebar 提供“专题路径库”，支持创建、改名、删除和搜索本地 Knowledge Pathway。
- 每条专题路径可以维护总览、标签和多份来源资料；同一份资料也可以加入多条专题路径。
- 资料库菜单支持将已有资料加入或移出专题路径；删除路径或资料时会同步清理本地关联记录。
- 专题路径详情页展示来源资料、知识节点数量与本地路径概览，并按概念、观点、证据、问题分类呈现节点。
- 专题路径可以批量管理已有知识卡片；同一张卡片也可以加入多条路径，删除卡片或路径时会同步清理关联。
- 单张卡片编辑器支持选择专题路径，因此阅读器选区 AI 制卡、批注制卡和摘要制卡都能在保存时直接归入专题。
- 专题路径详情支持轻量关系组织：可在知识节点间建立定义、支持、挑战、扩展和相关五类本地关系。
- 路径详情以清晰列表表达节点连接，并自动将观点节点与证据节点的“支持”关系整理为观点—证据链。
- 移出节点、删除卡片或删除专题路径时，相关本地关系会同步清理。
- 专题路径详情支持导出完整 Markdown 报告，包含专题总览、来源列表、文献 / 资料贡献矩阵、知识节点、节点关系、观点—证据链、学习路径建议和待补全问题。
- 知识节点支持轻量校准：待核验、已确认、需跟进三种状态，以及重点、已理解和校准备注。
- 专题路径详情会使用透明的本地规则提示结构缺口，包括来源不足、缺少概念或证据、未确认节点、未连接证据的观点和待补全问题。
- 专题 Markdown 报告会同步导出校准汇总与本地待补提示。
- 资料来源支持作者、年份、来源类型、网页链接、来源备注、可信度标记和主要贡献说明。
- 资料库支持添加网页资料：保存用户确认过的网页链接与正文摘录，并生成可阅读、可归类、可删除的本地 Markdown 副本。
- 专题路径详情可直接编辑来源资料，Markdown 报告会导出作者年份、可信度与升级后的文献贡献矩阵。
- 专题路径详情提供来源资料夹筛选，可按关键词、来源类型和可信度快速定位资料，并从来源卡片打开关联知识节点。
- 每份来源卡片会显示主要贡献与关联节点数量，便于核验资料在专题中的实际作用。
- 专题支持独立的“外部补全候选”区域：手动记录候选网页，阅读确认后再纳入正式来源。
- 本地缺口规则会提示待核验来源、来源元数据缺失、权威来源不足和仍待处理的外部补全候选。
- 专题 Markdown 报告会单独导出外部补全候选清单，不与正式来源混淆。
- 网页资料导入支持本地半自动提取：粘贴网页正文或 HTML 后，可提取标题、作者、年份和清洗后的正文摘录。
- 专题路径详情展示来源质量概览，包括元数据完整度、核验进度、权威来源数量和补全候选数量。
- 专题 Markdown 报告会导出来源质量概览，便于后续写作或核验时判断来源可靠性。
- 来源质量概览提供一键筛选入口，可直接跳转到“待核验来源”“缺作者年份来源”和“权威来源”的资料列表。
- 外部补全候选会根据本地规则显示处理建议，例如先补链接、补全作者年份、记录纳入理由或阅读后纳入路径。
- 专题 Markdown 报告和学习路径建议会同步列出待核验来源、元数据待补资料与候选处理建议，便于导出后继续执行。
- 专题路径详情提供本地“写作准备度”检查，提示观点是否有证据支撑、证据是否可回到来源、来源是否适合引用核验、问题和节点是否仍待处理。
- 专题 Markdown 报告会导出写作准备度检查结果，但不会一键生成论文。
- 专题路径详情提供本地“写作准备大纲”草稿，按问题界定、核心论点、证据材料和待处理问题整理已有节点与关系。
- 专题 Markdown 报告会导出写作准备大纲，作为写作前材料整理清单，不自动生成正文。
- 知识节点支持按节点类型与来源质量组合筛选，可快速定位待核验观点、待补证据等写作前优先处理对象。
- 专题路径详情和 Markdown 报告提供本地“写作行动清单”，把待补证据、待核验来源、候选资料和未校准节点整理为可执行任务。
- 写作行动清单中的可定位任务支持一键套用对应来源或节点筛选，也可直接跳转到外部补全候选区。
- 写作行动清单会预览相关节点、来源或候选资料标题，帮助用户在跳转前判断优先处理对象。
- TXT 与 Markdown 支持 UTF-8 文本预览、正文选择、摘录制卡和 AI 摘要。
- TXT 与 Markdown 阅读器支持正文搜索、命中高亮、匹配计数和前后跳转。
- PDF Reader 使用 `PDFKit.PDFView`，支持原生滚动、选择文本、翻页、缩放和当前页文本提取。
- PDF Reader 支持输入页码直达，并提供全文搜索、命中高亮、匹配计数和前后跳转。
- PDF Reader 工具栏突出“生成摘要”和“创建摘录卡片”，文本提取与缩放重置收纳在更多菜单中。
- PDF Reader 会持续保存上次阅读页码，再次打开资料时自动恢复阅读位置。
- 资料库显示未开始、阅读中、已完成状态，支持手动切换完成状态；最近阅读按最后打开时间排序。
- Dashboard 提供“继续阅读”入口，优先展示最近打开的资料。
- 首次空库启动会在本机自动载入一份 Markdown 示例材料和三张引导卡片；设置页与空状态支持恢复示例体验，重复载入不会创建副本。
- 扫描版 PDF 无法提取文本时，会提示当前 Demo 暂不支持 OCR。
- 知识卡片使用 SwiftData 保存在本机，支持新建、编辑、删除、搜索与标签筛选。
- 知识卡片页支持在卡片网格与主题分组之间切换，并可通过卡片菜单快速勾选或新建主题。
- 卡片列表、Dashboard 最近卡片和 Reader 右侧相关卡片支持打开完整详情，查看正文、标签、来源和更新时间。
- 带来源的知识卡片可以重新打开原文；PDF 卡片会回到保存时记录的页码，学习路径中的卡片也支持来源回溯。
- 知识卡片页支持将当前列表导出为 Markdown，保留卡片类型、来源、页码、标签和正文。
- PDF Reader 支持将当前选中文本连同来源文档和页码保存为摘录卡片。
- PDF 与文本阅读器支持对选中的名词、句子或片段一键生成 AI 卡片草稿，自动预填标题、类型、标签、归纳内容和原文选区，用户确认修改后再保存。
- Reader 右侧 Inspector 会根据当前选区查找已有资料、摘要和知识卡片中的相同线索，支持直接打开关联内容继续核验。
- 原文摘录入口继续保留在阅读器更多菜单中。
- PDF 与文本阅读器支持本地批注：选中原文后添加备注，Reader 右侧集中查看、编辑、删除并回到原文；PDF 使用页码锚点，文本资料回到对应选区首次出现的位置。
- 阅读器会使用淡紫色常驻标记批注原文；Reader 右侧支持搜索批注、按 PDF 当前页筛选，并将当前筛选结果导出为 Markdown。
- Reader 右侧每条批注支持一键生成 AI 卡片草稿：本地 Demo AI 会结合用户备注与原文选区预填标题、类型、标签和正文，用户确认修改后再保存。
- Reader 右侧支持勾选多条批注进行本地批量整理：轻量推断共同主题，生成概念卡、观点卡和证据卡三类可编辑草稿，用户统一确认后再写入卡片库。
- Dashboard 展示真实知识卡片数量和最近创建的卡片。
- Reader 右侧 Inspector 展示当前选区和当前文档的相关知识卡片。
- 测试版默认开启本地 Demo AI，无需 API Key 即可体验摘要、选区解释、卡片保存和 Markdown 导出。
- 本地 Demo AI 结果会明确标注为结构化示例，不会发送用户文本，也不代表真实模型分析结果。
- 设置页保留可选的 OpenAI-compatible Provider、Endpoint 和 Model 配置，用于后续真实 API 联调。
- 设置页提供 DeepSeek 预设，可使用用户自己的 DeepSeek API Key 调用真实模型，相关消耗由用户自己的模型服务账户承担。
- API Key 只保存在 macOS Keychain，不写入 `UserDefaults` 或 SwiftData，并支持一键删除。
- 设置页支持调用已配置的 OpenAI-compatible 服务测试连接。
- Reader 和右侧 Inspector 支持 AI 文档速读；超过 `12000` 字符时仅发送前 `12000` 字符并明确提示。
- PDF 和文本阅读器支持对当前选区进行 AI 概念解释；超过 `4000` 字符时仅发送前 `4000` 字符，并可保存为概念卡。
- AI 摘要可以保存为带来源文档的 `summary` 知识卡片。
- 完成摘要后，右侧 Inspector 会预览可编辑的概念卡、观点卡和证据卡草稿；用户逐张核验后再保存。
- 学习路径页会将卡片标签作为主题，按“理解概念、阅读观点、核验证据、复习卡片、导出行动”生成轻量路径。
- 学习路径支持导出一日舱 DayCabin 风格的本地 Markdown 任务草稿；当前为概念联动，尚未连接一日舱 API。
- 首次启动展示“参与知径 Knowvia 学习研究计划”说明；用户可以同意、暂不参与或稍后再说，提示只展示一次。
- 用户可以在设置页重新查看研究条款、主动参与或随时退出研究计划，选择仅保存在本机。
- 当前 Demo 不包含任何遥测 SDK 或研究数据上传。
- 学习路径已提供轻量主题归类和任务草稿导出；知识图谱和写作辅助保留为后续储备，不在 Sidebar 中作为当前重点展示。

在开启 App Sandbox 后，默认资料路径位于类似以下位置：

```text
~/Library/Containers/com.knowvia.Knowvia/Data/Library/Application Support/KnowviaLibrary/
```

## 项目结构

```text
Knowvia/
  App/          App 入口
  Models/       SwiftData 模型与本地偏好配置
  Views/        Dashboard、资料库、Reader 与 Inspector
  ViewModels/   导航、资料库导入与 Reader 状态
  Services/     文件导入、PDF 提取、文本预览、Keychain、AIService
  Components/   Logo、资料行、统计卡片、学习路径
  Utilities/    VI 色彩 Token、Hex 颜色支持、AI Prompt 模板
  Resources/    App Icon 与 Accent Color
KnowviaTests/   导入、持久化、PDF 提取、偏好配置、Keychain、Prompt 与 AIService 测试
Scripts/        App Icon 生成脚本
prototype-web/  前期网页视觉探索稿
```

## VI 与图标

应用使用 `Concept A - Path-based Academic Minimalism` 作为主视觉，并在 Dashboard 学习路径中少量吸收 Concept D 的暖色。Logo 与 App Icon 更新为轨道、路径与知识光点组合图形：浅色界面直接使用品牌指南中的标准白底彩色 Logo 资产，深靛蓝 App Icon 使用对应的彩色紧凑版本。

重新生成 App Icon：

```bash
swift Scripts/generate_app_icon.swift \
  Knowvia/Resources/Assets.xcassets/AppIcon.appiconset
```

## 当前阶段边界

近期开发聚焦“专题 Knowledge Pathway 生成”闭环：创建专题路径、多源资料归档、知识节点整理、来源追溯、轻量校准、路径视图与导出。当前尚未包含 Notion 导出、一日舱 API 真实联动、OCR、云同步、账号、社区、付费功能或任何研究数据上传。复杂全局知识图谱、自动无限联网检索、完整音视频处理和大型后端平台仍不属于 MVP。
