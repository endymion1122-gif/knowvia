# 知径 Knowvia Knowledge Pathway 开发落地差异说明

## 1. 当前正式需求基线

自《知径 Knowvia 产品正式档案》纳入仓库后，产品定位更新为：

> 知径 Knowvia 是一个 AI 增强型多源知识路径生成系统，帮助用户从论文、网页、书籍、课程音视频、笔记与可靠外部来源中生成可追溯、可校准、可学习、可输出、可持续生长的个人 Knowledge Pathway。

早期“AI 阅读与知识卡片工具”不再作为最终定位，但其中已经完成的能力继续作为基础层复用。

## 2. 主线变化

### 旧主线

导入单篇资料 -> 阅读 -> AI 摘要 -> 摘录与批注 -> 知识卡片 -> 轻量学习路径

### 新主线

创建专题路径 -> 汇入多源资料 -> 解析来源 -> 抽取知识节点 -> 组织节点关系 -> 用户轻量校准 -> 生成 Knowledge Pathway -> 卡片沉淀 -> 输出迁移 -> 路径持续生长

核心产品对象从“单篇资料和卡片”升级为“专题 Knowledge Pathway”。

## 3. 已有能力复用映射

| 已实现能力 | 在新架构中的位置 |
| --- | --- |
| PDF / TXT / Markdown 导入 | 多源资料夹基础层 |
| PDF 与文本阅读器 | 来源核验与回到原文 |
| 阅读搜索、页码恢复 | 来源定位 |
| 批注、高亮、筛选与导出 | Reflection Node 个人理解节点 |
| 选区 AI 制卡 | Concept / Claim / Evidence Node 初稿 |
| 批注转卡片与批量整理 | 用户轻量校准与卡片沉淀 |
| 卡片来源回溯 | 证据来源追溯 |
| 卡片主题分组 | 专题路径的早期主题标签 |
| 轻量学习路径 | Knowledge Pathway 的早期输出视图 |
| 本地 Demo AI 与 OpenAI-compatible 配置 | 本地优先、模型无关架构起点 |

## 4. 下一阶段开发顺序

### Phase A：专题路径库

1. 新增 `KnowledgePathway` 本地模型。
2. 支持创建、改名、删除和查看专题路径。
3. 允许将已有资料归入一条或多条专题路径。
4. Dashboard 从“资料总览”逐步转为“路径总览”。

### Phase B：路径详情页

1. 增加专题总览、资料来源、知识节点和待补全问题区。
2. 将现有卡片与批注挂接到专题路径。
3. 初步区分 Concept、Claim、Evidence、Reflection 和 Source 节点。
4. 支持保留、删除、改名、标重点、回到来源等轻量校准操作。

### Phase C：轻量关系组织

1. 增加 `KnowledgeRelation` 本地模型。
2. 首批关系支持 `defines`、`supports`、`challenges`、`extends`、`related_to`。
3. 用清晰列表和简单路径视图表达关系，不做复杂全局知识图谱。
4. 增加观点与证据的对应视图。

### Phase D：路径导出

1. 导出专题 Markdown 报告。
2. 导出来源列表和文献贡献矩阵。
3. 导出学习路径和待补全问题。
4. 保留一日舱任务草稿概念联动。

### Phase E：轻量校准与缺口提示

1. 为知识节点增加待核验、已确认和需跟进状态。
2. 支持标重点、标记已理解和保存校准备注。
3. 用透明的本地规则提示来源、结构、证据与问题缺口。
4. 将校准汇总和缺口提示纳入专题报告。

### Phase F：来源元数据与网页资料导入

1. 为来源资料增加作者、年份、来源类型、网页链接、备注、可信度与主要贡献字段。
2. 支持手动添加网页资料，将用户确认过的正文摘录保存为本地 Markdown 副本。
3. 在资料库和专题路径详情中编辑来源元数据。
4. 将作者年份、可信度与主要贡献纳入专题来源列表和文献贡献矩阵。

### Phase G：来源资料夹与外部补全候选

1. 在专题详情中按关键词、来源类型和可信度筛选正式来源。
2. 在来源卡片中展示主要贡献、关联节点数量，并支持直接打开关联节点。
3. 增加独立的外部补全候选区；候选资料需经用户确认后才能纳入正式来源。
4. 用本地规则提示待核验来源、元数据缺失、权威来源不足与待处理候选。
5. 在专题 Markdown 报告中单独导出外部补全候选清单。

### Phase H：网页半自动提取与来源质量概览

1. 网页资料导入支持粘贴正文或 HTML 后，本地提取标题、作者、年份与可读正文摘录。
2. 保持用户确认优先，不做自动无限联网抓取。
3. 专题详情增加来源质量概览，显示元数据完整度、核验进度、权威来源数量和补全候选数量。
4. 专题 Markdown 报告同步导出来源质量概览。

## 5. MVP 暂不开发

- 复杂全局知识图谱
- 自动无限联网检索
- 完整音视频处理
- 云同步与账号系统
- 社区与多用户协作
- 一键生成论文
- 复杂学习分析看板
- 大型后端平台

## 6. 下一批具体任务

Phase A、Phase B、Phase C、Phase D、Phase E、Phase F、Phase G、Phase H、Phase I 已完成。

### Phase I：来源质量处理闭环

1. 增加专题来源质量筛选入口，可一键查看“待核验来源”“缺作者年份来源”和“权威来源”。
2. 为外部补全候选生成更清晰的本地处理建议，例如先补链接、补全作者年份、记录纳入理由或阅读后纳入路径。
3. 支持从来源质量概览直接跳转到需要补全的资料列表。
4. 专题 Markdown 报告和学习路径建议同步导出待核验来源、元数据待补资料与候选处理建议。
5. 继续保持本地优先，不启动自动无限联网检索。

Phase J 已启动：

1. 增加专题路径的“写作准备度”本地检查，例如观点是否有证据、来源是否可引用、问题是否仍未处理。（已完成基础版）
2. 在专题报告中生成更适合写作前检查的本地大纲草稿，但不提供一键生成论文。（已完成基础版）
3. 支持按知识节点类型和来源质量组合筛选，帮助用户优先处理关键观点与关键证据。（已完成基础版）

Phase K 已启动：

1. 增加本地“写作行动清单”，把写作准备度、来源质量、候选资料和节点校准结果整理为可执行任务。（已完成基础版）
2. 行动清单支持一键定位到对应来源筛选、节点筛选或外部候选区，减少从提示到处理之间的跳转成本。（已完成基础版）
3. 行动清单展示相关节点、来源或候选资料标题，帮助用户先判断处理对象再跳转。（已完成基础版）
